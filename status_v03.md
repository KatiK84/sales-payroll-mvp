# Status — sales-payroll-mvp — v0.3

**Дата:** 18.03.2026
**Этап:** v0.3 — Обновлённая логика фиксированной части

## Что сделано

### Новая логика групп (обновлено vs v0.2)
- **Группа "удалённый менеджер"**: теперь выплачивается пониженный фикс даже при недостижении порога:
  - полная занятость: ≥10 000 → 800, иначе → 400 (было: → 0)
  - частичная занятость: ≥5 000 → 400, иначе → 200 (было: → 0)

### Новый режим "вручную"
- Поддержан режим расчёта `вручную`:
  - задаётся `ручной_фиксированный_оклад` (число)
  - задаётся `минимальный_порог` (число или null/"нет")
  - если порог есть: выплата только при достижении порога
  - если порога нет: выплачивается всегда

### Новая схема справочника managers.json
Поля: `manager_name`, `режим_фикса`, `группа_фикса`, `тип_занятости`, `ручной_фиксированный_оклад`, `минимальный_порог`
Режимы: `по группе` / `вручную`
Группы: `удалённый менеджер` / `фикс независимо от поступлений` / `без оклада`

### Интерфейс — новый реестр
Таблица фиксированной части с колонками:
- Менеджер, Режим фикса, Группа фикса, Тип занятости
- База поступлений EUR, Мин. порог, Фиксированный оклад EUR, Результат правила

### Обработка ошибок
Явный блок ошибок для:
- не найден в справочнике
- не указан режим_фикса
- вручную: не указан оклад
- вручную: порог не число и не null
- по группе: не указана группа/тип занятости

## Поля, использованные в расчёте
