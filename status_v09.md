# Sales Payroll MVP — Status v0.9
**Дата:** 18.03.2026  
**Файл:** `index.html` (standalone, ~92KB)

## Что сделано в v0.9

### 1. Рабочий цикл периода
- Выбор месяца и года в блоке "Обзор" (селект + input)
- При загрузке файла период подставляется автоматически из даты отчёта
- Статус периода: `✏ Черновик` / `✓ Сохранён` / `Утверждён`
- Предупреждение при попытке перезаписать уже сохранённый период

### 2. Сохранение периода (savePeriod)
Snapshot включает:
- Зарплатный итог по каждому менеджеру (фикс, бонус, провизион, итого, правила)
- Список сделок, вошедших в провизион (`provDeals`)
- Список дублей (`dupDeals`)
- Список карантинных сделок (`quarDeals`) + ручная проверка (`manualDeals`)
- Список исключённых по платформе (`exclDeals`)
- Ошибки расчёта (`errors`)
- Метаданные: файл, период, дата сохранения

### 3. Защита от повторного начисления
- Реестр учтённых сделок: `localStorage['spm_ddDedup']` — Set из ключей `Bestellnummer|Rechnungsnummer|Verkaeufer`
- При сохранении периода все сделки провизиона регистрируются в реестре
- При следующей загрузке JTL — дубли автоматически получают статус `дубль` с указанием периода: _«Уже учтена в провизионе (Август 2025)»_
- Кнопка "Очистить реестр учтённых сделок" — сбрасывает реестр и восстанавливает статус сделок

### 4. История периодов
- Хранится в `localStorage['spm_hist']` (до 24 периодов)
- Карточки с бейджами статуса, суммами, количеством сделок/дублей
- Кнопка "✓ Утвердить" переводит период в финальный статус

### 5. Детализация периода (5 вкладок)
- **💰 Зарплата** — таблица по менеджерам: база поступлений, база провизиона, фикс, бонус, провизион, итого
- **✅ Сделки в провизионе** — все учтённые сделки с Betrag и совместными долями
- **🔄 Дубли** — исключённые как повторные с причиной
- **⏳ Карантин/Ручная проверка** — сделки, не прошедшие проверку
- **🚫 Исключены** — по платформе

## Хранение данных (localStorage)

| Ключ | Содержимое |
|------|-----------|
| `spm_mgr` | Справочник менеджеров с настройками |
| `spm_dedup` | Реестр учтённых строк оплат (фикс/бонус) |
| `spm_ddDedup` | Реестр учтённых сделок провизиона |
| `spm_hist` | История расчётных периодов (до 24) |
| `spm_deals` | Текущий JTL-реестр сделок |
